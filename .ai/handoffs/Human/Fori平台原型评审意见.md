# Fori平台原型评审意见

当前原型从功能设计、版面美化设计到交互体验存在巨大缺失！请按照以下要求，调用claude code和 codex交叉作业，互相提议-设计-评审-定稿，进行完善后再提交一版原型设计、技术方案及产品开发项目管理及任务分配方案：

1. 对照初始需求，完整梳理并维护初始需求中提及的**核心目标与模块功能**，不得有任何遗漏，形成一份完整功能清单；对各功能所属模块及功能本身进行分级，**不得删减任何功能**，仅可对功能进行丰富；站在使用者角度进行推导，使功能与模块清单更加完善，并持续进行版本迭代。
    
2. 从使用者角度出发，使用者包括：经纪人、平台工作人员、意向购房者、意向卖房者；其中购房者与卖房者常为同一人，存在改善性需求；针对所有页面呈现、使用体验，从使用体验角度推导页面应呈现内容、功能排布方式、交互实现方式；维护一份**所有使用角色的功能与交互清单**，从使用体验角度出发进行推导。版面的引导词、视觉感观及交互逻辑都必须针对使用者进行专门设计。
    
3. 平台核心模块-房源字典功能缺失，需补充完善：
    
    - 房屋房源词典（房源字典）需支持两种呈现方式：**地图式呈现、卡片及列表清单式呈现**；
    - 针对不同使用者，房源信息需做**差异化呈现**；
    - 部分房源信息需进行**保密、隔离处理**。
    - 参照日本 SUUMO 网站对房源信息披露进行规范化、模板化设计。

4. 对平台**共建共赢裂变机制**进行深入、详细的设计并实现：
    
    - 包含经纪人初始录入、后续修订补充，以及业主、购房人、平台管理员参与房源信息维护、评价管理等，平台需建立一套完整的共建共赢维护逻辑；
    - 当房地产交易达成时，需对房源信息推荐、信息传播、媒体制作、带看服务、交易全程服务等各环节的利益进行清晰核算与分配；
    - 在保证信息公正、规范的前提下，最大化减少房屋信息获取及交易过程中的摩擦，相关机制需完整设计。
    
5. 对整个设计开发过程进行完整管理与分析：
    
    - 对人类用户指令、多智能体协作过程进行完整分析；
    - 对项目中文档、脚本与现行做法不一致或存在矛盾的内容，实施版本管理，建立明确机制区分哪些文档当前有效；
    - 核心目的：为后续同类项目在产品设计与开发时提供借鉴；
    - 在本项目及整体环境中，长期维护并持续迭代：设计开发经验集、技能集、需求集（人类用户指令集）。

6. 对房源交易价格评估及撮合机制进行深入研究，并完成功能实现设计，作为平台核心功能之一。定价与撮合需站在公正、中立、客观、理性的角度实现，同时在心理层面与实际利益层面最大化保全、照顾各参与方，促使交易过程快速达成，需先形成完整方案再进行功能实现。
    
7. 本平台为 Agent 原生项目产品，在现有原型基础上，每一页、每一项功能均需考虑如何基于当前暂定的 OpenClaw Agent 框架实现，包括各类使用者在平台上的交互方式（语音、文字、拍摄等输入方式）需完善支持，并对输出结果进行设计。

8. 对经纪人和平台的收费获益体系进行完整设计：

- 用户注册登录分级设计：未验证登录可查看的信息范围、验证登录后可查看的信息范围均需明确设计；一般性查询仅需电话号码验证登录；更多真实信息则按必要最小、易实现的方式实现身份验证。
- 付费机制设计：用户获得有价值服务时需付费，付费方式需完整设计。
- 利益分配设计：对平台各方参与者的利益获得方式与利益分配规则进行设计，在公开可信、各方觉得值得的前提下达成博弈平衡，顺畅实现交易、付费与应得利益分成；包含机制设计、功能设计及交互实现设计。
