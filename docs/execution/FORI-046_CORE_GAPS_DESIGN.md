## 1. Gap 1 — 地图核心房源字典

### 1.1 设计原则

地图不是「找房」Tab 的侧边功能页，而是**楼盘字典的核心主入口**。用户初次接触 Fori，应从地图直观感知全国楼盘数据分布；搜索、筛选、钻取都从地图发起，而不是先看列表再跳地图。

### 1.2 路由体系重新定义

```
/explore/dict                     ← 入口页，默认渲染地图模式（改变默认 Tab）
  切换 Tab: [地图 ★默认] [卡片] [列表]

/explore/map                      ← 保留为独立地图全屏页（deep link 入口）
/explore/dict/[communityId]       ← 小区详情（从地图 pin 点击钻取）
```

**变更说明**：`/explore/dict` 默认选中「地图」Tab（现在默认「卡片」），地图成为用户进入字典的第一视图。

### 1.3 地图技术栈决策

#### 原型（不需要 API Key）

| 层 | 技术 | 说明 |
|----|------|------|
| 地图库 | `leaflet` + `react-leaflet` | 开源，无 API key，CDN 可用 |
| 底图瓦片 | OpenStreetMap (openstreetmap.org) | 免费，用于原型 |
| 小区标记 | `react-leaflet` Marker + 自定义 DivIcon | 渲染价格气泡 |
| 聚合 | `leaflet.markercluster` 或自实现计数气泡 | 城市级缩放时聚合 |
| Mock 数据 | `/lib/mock-map.ts` — 50 个小区 GeoJSON | 覆盖 8 个城市 |

依赖安装（Codex 实现时执行）：
```bash
npm install leaflet react-leaflet @types/leaflet
# 可选聚合：
npm install leaflet.markercluster @types/leaflet.markercluster
```

**原型底部注释**（每个地图页面必须显示）：
> 「原型使用 OpenStreetMap + Mock 坐标；生产接入高德地图 JS API 2.0 + 开源底图业务数据层」

#### 生产（高德 · 主选 + 开源业务层）

```
生产地图架构：
  底图层：   高德地图 JS API 2.0（DECIDED · 主选）；腾讯地图（备选 fallback）
             ADR 已决定：高德在中国大陆合规性最佳、POI 数据最全、JS SDK 最成熟
             非阻塞后续事项：高德 Web 服务 Key 申请（由工程团队在部署阶段完成）
  业务层：   平台自维护楼盘 GeoJSON 叠加层（~80 万小区坐标）
  聚合层：   服务端地理空间聚合（Tile-based clustering，PostGIS）
  实时层：   在售套数、价格更新 SSE 推送到地图
```

备注：800 万级小区数据不应全量下发前端；生产实现必须基于 viewport bounds 动态拉取。

## 2. Gap 2 — 短视频素材制作与自媒体触达

### 2.1 现状分析

已有（不重复实现）：
- `/marketing/generate`：素材生成（poster/script/description/cover），风格/受众/重点参数，历史记录
- `/marketing/manage`：推广计划管理，曝光/点击/转化聚合指标

**缺失（本次补齐）**：
1. 短视频专属制作流程（分镜脚本 → 视频参数 → 预览态）

## 附录 A — 设计决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 原型地图库 | `leaflet` + `react-leaflet` | 无 API key；react 集成好；CDN 可用；易于 Codex 实现 |
| 不用 maplibre-gl | 跳过 | maplibre 体积更大，原型阶段不需要 Vector tiles |
| **生产地图供应商** | **高德地图 JS API 2.0（DECIDED）** | 中国大陆合规性最佳；POI 数据全；JS SDK 成熟；腾讯地图为备选 fallback |
| 价格单位 | `priceMin/priceMax` 存元/㎡，筛选用参考总价（万元）| 元/㎡ 是行业标准字段；总价 = 单价 × avgAreaSqm ÷ 10000，用于用户感知的总价筛选 |
| 区域筛选 | 一级筛选（城市下的行政区）| 区域是用户找房最核心的地理维度；与城市为父子关系，切换城市自动重置 |
| 短视频路由 | `/marketing/video`（独立路由）| 短视频制作流程够复杂，不适合塞入现有 generate 页 |
| CRM 路由 | 新增 `/leads`，不改造现有 buyers | 保持向后兼容；buyers 保留为快速视图 |
| 房东线索独立路由 | `/workspace/agent/landlords` | 房东漏斗阶段与买家完全不同，共用列表会混乱 |
| FunnelBar 不用 ECharts | 纯 CSS 进度条 | 漏斗展示不需要交互图表；减少依赖复杂度 |
